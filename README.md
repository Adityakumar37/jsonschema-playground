# JSON Schema Playground

A fast, browser-based JSON Schema validator built with Rust, WebAssembly, and React.

Paste a JSON Schema on the left, paste your JSON on the right, and validate instantly in the browser. No server. No backend. Everything runs locally via WebAssembly.

---

## Built With

- [jsonschema](https://github.com/Stranger6667/jsonschema) — High-performance Rust JSON Schema validation library by Dmitry Dygalo, licensed under MIT
- [wasm-pack](https://rustwasm.github.io/wasm-pack/) — Compiles Rust to WebAssembly
- [React](https://react.dev) — Frontend UI
- [Vite](https://vitejs.dev) — Frontend build tool

## Project Structure

```
jsonschema-playground/
├── wasm-validator/          Rust library that wraps jsonschema and exposes validate() to JavaScript
│   ├── src/
│   │   └── lib.rs           Core validation logic written in Rust
│   ├── pkg/                 Auto-generated by wasm-pack, contains the compiled WASM and JS wrapper
│   └── Cargo.toml           Rust dependencies including jsonschema = "0.46"
└── frontend/                React application
    ├── src/
    │   ├── pkg/             Copy of wasm-validator/pkg, loaded by the React app
    │   ├── App.jsx          Main UI component with schema and JSON editors
    │   └── main.jsx         React entry point
    ├── index.html           Root HTML file
    ├── vite.config.js       Vite configuration with WASM plugin
    └── package.json         Node dependencies
```

---

## How It Works

1. The `jsonschema` Rust library is added as a dependency in `Cargo.toml`
2. `lib.rs` wraps the library's `validate()` function and exposes it to JavaScript using `wasm-bindgen`
3. `wasm-pack` compiles the Rust code into a `.wasm` binary and generates a JavaScript wrapper
4. The React frontend loads the WASM file at runtime using the `vite-plugin-wasm` plugin
5. When the user clicks Validate, React calls the Rust function directly in the browser
6. Errors are returned as a formatted string and displayed in the UI

No network requests are made during validation. Everything runs in the browser.

## Prerequisites

Make sure you have the following installed before running the project locally.

- [Rust](https://rustup.rs) — Install via rustup
- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/) — Compiles Rust to WASM
- [Node.js](https://nodejs.org) — Version 18 or higher

---

## Running Locally

### Step 1 — Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/jsonschema-playground
cd jsonschema-playground
```

### Step 2 — Build the WASM binary

```bash
cd wasm-validator
wasm-pack build --target web
```

This compiles the Rust code and generates the `pkg/` folder.

### Step 3 — Copy the generated package to the frontend

```bash
cd ../frontend
cp -r ../wasm-validator/pkg ./src/pkg
```

### Step 4 — Install frontend dependencies

```bash
npm install
```

### Step 5 — Start the development server

```bash
npm run dev
```

Open `http://localhost:5173` in your browser.

---

## Rebuilding After Changes

If you modify `lib.rs` or `Cargo.toml`, you need to rebuild the WASM and copy it again.

```bash
cd wasm-validator
wasm-pack build --target web
cd ../frontend
cp -r ../wasm-validator/pkg ./src/pkg
```

Then refresh the browser.
